KEY ARCHITECTURAL NOTES:
- modules/ follow the pattern: router (HTTP) в†’ service (business logic) в†’ Prisma (DB)
- engines/ are pure TypeScript functions with no Express/Prisma dependencies вЂ” easily unit tested
- All secrets validated at startup via Zod in config/env.ts вЂ” no runtime surprises
- AuditLog entries written by requestLogger.ts middleware вЂ” no manual logging in services
- Redis used for: rate limiting, cohort cache (TTL 1hr), LLM response cache (TTL 24hr)
- Prisma client instantiated once in config/prisma.ts and imported everywhere (singleton)
